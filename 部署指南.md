# 企业微信会议录音转写系统 - CentOS 7 部署指南

## 服务器环境信息
- MySQL: 本地连接（localhost:3306）
- 端口: 8090（8080已被pin-server占用）
- 进程管理: Supervisor

## 部署步骤

### 1. 创建目录结构
```bash
mkdir -p /opt/wechat_meeting
mkdir -p /opt/wechat_meeting/uploads
mkdir -p /opt/wechat_meeting/dist
mkdir -p /var/log/wechat_meeting
mkdir -p /var/log/meeting_meeting
```

### 2. 上传文件到服务器
在本地执行：
```bash
# 上传jar包
scp meeting-transcribe-1.0.0.jar root@你的服务器IP:/opt/wechat_meeting/

# ⚠️ 重要：修改配置文件 application-prod.yml
# 需要修改以下配置项：
#   - password: MySQL密码
#   - jwt.secret: JWT密钥
#   - server.port: 8090（已配置）
#   - app.frontend-url: https://bjcastapp.bijienetworks.com
#   - app.backend-url: https://bjcastapp.bijienetworks.com
#   - file.upload-path: /opt/wechat_meeting/uploads/

# 上传配置文件（修改后的生产配置）
scp application-prod.yml root@你的服务器IP:/opt/wechat_meeting/application.yml

# 上传数据库初始化脚本
scp database_init.sql root@你的服务器IP:/opt/wechat_meeting/

# 上传supervisor配置
scp supervisord-meeting.ini root@你的服务器IP:/etc/supervisord.d/meeting-server.ini

# 上传前端打包文件（整个dist目录）
scp -r dist/* root@你的服务器IP:/opt/wechat_meeting/dist/
```

### 3. 配置MySQL数据库
在服务器上执行：
```bash
# 登录MySQL
mysql -u root -p

# 执行数据库初始化
source /opt/wechat_meeting/database_init.sql

# 或者手动执行
CREATE DATABASE IF NOT EXISTS meeting_recording DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE meeting_recording;
# 然后执行 database_init.sql 中的建表语句
```

### 4. 配置应用权限
```bash
chown -R root:root /opt/wechat_meeting
chmod -R 755 /opt/wechat_meeting
chmod 755 /opt/wechat_meeting/uploads
chown -R root:root /var/log/wechat_meeting
```

### 5. 配置Supervisor
```bash
# 复制supervisor配置
cp /opt/wechat_meeting/supervisord-meeting.ini /etc/supervisord.d/meeting-server.ini

# 重新加载supervisor配置（两种方式任选其一）
# 方式一：推荐 - 不会影响其他运行中的服务
supervisorctl reread
supervisorctl update
supervisorctl start meeting-server

# 方式二：会重启整个supervisor，可能短暂影响其他服务
# supervisorctl reload
# supervisorctl start meeting-server

# 查看状态
supervisorctl status meeting-server
```

### 6. 检查应用启动日志
```bash
tail -f /var/log/wechat_meeting/meeting-server.log
```

### 7. 验证应用
```bash
# 测试健康检查
curl http://localhost:8090

# 或访问
http://你的公网IP:8090
```

### 8. 配置防火墙（如果需要）
```bash
# 开放8090端口
firewall-cmd --permanent --add-port=8090/tcp
firewall-cmd --reload

# 或者直接关闭防火墙（不推荐）
systemctl stop firewalld
systemctl disable firewalld
```

## 重要配置说明

### 1. 配置文件修改清单（application-prod.yml）
上传前必须修改以下配置项：

```yaml
# 1. MySQL 密码
password: 你的MySQL密码

# 2. JWT 密钥（建议使用随机字符串）
jwt:
  secret: 你的JWT密钥

# 3. 前后端访问地址（已配置为HTTPS）
app:
  frontend-url: https://bjcastapp.bijienetworks.com
  backend-url: https://bjcastapp.bijienetworks.com
```

### 2. 数据库连接
确保MySQL服务正在运行：
```bash
systemctl status mysqld
```

如果MySQL未运行：
```bash
systemctl start mysqld
systemctl enable mysqld
```

### 3. 修改MySQL密码（如需要）
```bash
mysql -u root -p
ALTER USER 'root'@'localhost' IDENTIFIED BY '新密码';
FLUSH PRIVILEGES;
```

### 4. 文件上传目录
确保 uploads 目录有足够的写入权限：
```bash
chmod 777 /opt/wechat_meeting/uploads
```

### 5. 企业微信配置
在企业微信管理后台需要更新：
- 应用回调URL: `https://bjcastapp.bijienetworks.com/api/wechat/callback`
- ⚠️ 必须使用 HTTPS，不能使用 HTTP

## 故障排查

### 应用启动失败
1. 检查端口是否被占用：
```bash
netstat -tlnp | grep 8090
```

2. 检查Java是否安装：
```bash
java -version
```

3. 检查配置文件语法：
```bash
cat /opt/wechat_meeting/application.yml
```

### 数据库连接失败
1. 检查MySQL服务：
```bash
systemctl status mysqld
```

2. 测试数据库连接：
```bash
mysql -u root -p -h localhost meeting_recording
```

### 日志显示权限问题
```bash
chown -R root:root /opt/wechat_meeting
chmod -R 755 /opt/wechat_meeting
```

## Nginx反向代理配置（必须，使用HTTPS）

### 1. 准备SSL证书
你的泛域名证书（*.bijienetworks.com）可以直接使用：
- 证书路径：`/etc/nginx/ssl/fullchain.pem`
- 私钥路径：`/etc/nginx/ssl/private.key`

### 2. 上传Nginx配置
```bash
# 备份原有配置（如果有）
cp /etc/nginx/conf.d/bjcastapp.conf /etc/nginx/conf.d/bjcastapp.conf.bak

# 创建新的合并配置文件
cat > /etc/nginx/conf.d/bjcastapp.conf << 'EOF'
server {
    listen 443 ssl;
    server_name bjcastapp.bijienetworks.com;

    ssl_certificate      /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key  /etc/nginx/ssl/private.key;

    ssl_session_cache    shared:SSL:1m;
    ssl_session_timeout  5m;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    access_log /var/log/nginx/bjcastapp.access.log;
    error_log /var/log/nginx/bjcastapp.error.log;

    # 会议转录系统前端
    location / {
        root /opt/wechat_meeting/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 会议转录系统后端 API
    location /api/ {
        proxy_pass http://127.0.0.1:8090;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        client_max_body_size 500M;
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
    }

    # 现有服务 - castinfoprovider
    location ^~ /castinfoprovider/ {
        proxy_pass http://127.0.0.1:8009/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        client_max_body_size 1024m;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header Host $http_host;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        root /opt/wechat_meeting/dist;
        expires 30d;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    # 上传文件目录
    location /uploads/ {
        alias /opt/wechat_meeting/uploads/;
        autoindex off;
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name bjcastapp.bijienetworks.com;
    return 301 https://$server_name$request_uri;
}
EOF
```

### 3. 重启Nginx
```bash
# 测试配置语法
nginx -t

# 重启Nginx
systemctl restart nginx

# 查看状态
systemctl status nginx
```

### 4. 验证HTTPS访问
访问：`https://bjcastapp.bijienetworks.com`

### 5. 访问地址说明
- 前端地址：`https://bjcastapp.bijienetworks.com`
- 后端API：`https://bjcastapp.bijienetworks.com/api`
- 现有服务：`https://bjcastapp.bijienetworks.com/castinfoprovider/`

## 注意事项

1. **备份配置文件**: 部署前备份原有的配置文件
2. **日志管理**: 定期清理日志，避免磁盘空间不足
3. **监控**: 建议配置监控报警，及时发现服务异常
4. **数据备份**: 定期备份MySQL数据库
5. **安全**: 修改默认密码，使用复杂的JWT密钥
