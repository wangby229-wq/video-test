## 第一部分：MySQL 数据库配置

### 1.1 测试 MySQL 连接
```bash
/usr/local/mysql/bin/mysql -u root -p
```
**输入你的 MySQL root 密码**

如果不知道密码，需要重置：
```bash
/usr/local/mysql/bin/mysqladmin -u root shutdown
/usr/local/mysql/bin/mysqld_safe --skip-grant-tables &
sleep 5
/usr/local/mysql/bin/mysql -u root
```
在 MySQL 中执行：
```sql
FLUSH PRIVILEGES;
ALTER USER 'root'@'localhost' IDENTIFIED BY '你的新密码';
FLUSH PRIVILEGES;
EXIT;
```
然后重启 MySQL：
```bash
/usr/local/mysql/bin/mysqladmin -u root -p你的新密码 shutdown
/usr/local/mysql/bin/mysqld_safe &
```

### 1.2 创建数据库
登录 MySQL 后执行：
```sql
CREATE DATABASE IF NOT EXISTS meeting_recording DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
SHOW DATABASES;
EXIT;
```

## 第四部分：服务器上操作

SSH 登录到服务器：
```bash
ssh ecs-user@47.96.21.34
sudo su -
```

### 4.1 创建目录结构
```bash
mkdir -p /opt/wechat_meeting/uploads
mkdir -p /opt/wechat_meeting/dist
mkdir -p /var/log/wechat_meeting
chmod 777 /opt/wechat_meeting/uploads
```

### 4.2 移动文件
```bash
mv /tmp/meeting-transcribe-1.0.0.jar /opt/wechat_meeting/
mv /tmp/application.yml /opt/wechat_meeting/
mv /tmp/database_init.sql /opt/wechat_meeting/
rm -rf /opt/wechat_meeting/dist/*
mv /tmp/dist/* /opt/wechat_meeting/dist/
```

### 4.3 初始化数据库（如果需要）
```bash
/usr/local/mysql/bin/mysql -u root -p < /opt/wechat_meeting/database_init.sql
```

### 4.4 配置 Supervisor
创建文件：`/etc/supervisord.d/meeting-server.ini`
```ini
[program:meeting-server]
directory=/opt/wechat_meeting
command=/usr/bin/java -Xms256m -Xmx512m -jar meeting-transcribe-1.0.0.jar --spring.config.location=/opt/wechat_meeting/application.yml
numprocs=1
autostart=true
autorestart=true
startsecs=10
startretries=3
exitcodes=0
stopsignal=TERM
stopwaitsecs=60
user=root
redirect_stderr=true
stdout_logfile=/var/log/wechat_meeting/meeting-server.log
stdout_logfile_maxbytes=50MB
stdout_logfile_backups=10
```

### 4.5 启动服务
```bash
supervisorctl reread
supervisorctl update
supervisorctl start meeting-server
supervisorctl status meeting-server
```

### 4.6 检查日志
```bash
tail -f /var/log/wechat_meeting/meeting-server.log
```

## 第五部分：配置 Nginx

### 5.1 备份现有配置
```bash
cp /etc/nginx/conf.d/bjcastapp.conf /etc/nginx/conf.d/bjcastapp.conf.bak
```

### 5.2 修改 Nginx 配置
编辑文件：`/etc/nginx/conf.d/bjcastapp.conf`

将原有的 server 块替换为：

```nginx
server {
    listen 443 ssl;
    server_name bjcastapp.bijienetworks.com;

    ssl_certificate      /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key  /etc/nginx/ssl/private.key;

    ssl_session_cache    shared:SSL:1m;
    ssl_session_timeout  5m;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    access_log /var/log/nginx/bjcastapp.access.log;
    error_log /etc/nginx/bjcastapp.error.log;

    # 会议转录系统前端
    location / {
        root /opt/wechat_meeting/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 会议转录系统后端 API
    location /api/ {
        proxy_pass http://127.0.0.1:8090;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        client_max_body_size 500M;
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
    }

    # 现有服务 - castinfoprovider
    location ^~ /castinfoprovider/ {
        proxy_pass http://127.0.0.1:8009/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        client_max_body_size 1024m;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header Host $http_host;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        root /opt/wechat_meeting/dist;
        expires 30d;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    # 上传文件目录
    location /uploads/ {
        alias /opt/wechat_meeting/uploads/;
        autoindex off;
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name bjcastapp.bijienetworks.com;
    return 301 https://$server_name$request_uri;
}
```

### 5.3 重启 Nginx
```bash
nginx -t
systemctl restart nginx
systemctl status nginx
```
